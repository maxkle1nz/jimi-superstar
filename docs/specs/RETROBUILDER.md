# RETROBUILDER

RETROBUILDER is one canonical work method of JIMI SUPERSTAR.

It is also a general build doctrine that can be used beyond JIMI, but inside this house it should be treated as an explicit operational method rather than just background philosophy.

## Definition

Build the system backward from a fully grounded finished program in the field of ideas, then implement from constitutional center outward.

RETROBUILDER also allows architectural steering during implementation, as long as that steering is grounded, explicit, and reflected back into the constitutional map of the project.

## Why

This house touches:

- multi-engine execution
- policy
- approvals
- runtime state
- memory
- installable agent capsules

That is too much surface area for improvisational scaffolding.

RETROBUILDER works especially well when the system is alive enough to expose truth during the run.

That allows:

- inside-run architectural steering
- donor-system absorption with lower confusion
- continuous re-grounding without losing the build thread
- clearer integration of external subsystems into the house grammar

## Sequence

1. define the finished system
2. define contracts and invariants
3. define states and flows
4. define persistence and security
5. scaffold the smallest sovereign center
6. expand outward without breaking the center

## Inside-Run Steering

A core RETROBUILDER characteristic is that steering does not need to stop when code begins.

The method explicitly permits:

- pausing mid-run
- reassessing architectural truth
- changing the integration plan
- absorbing donor systems
- updating the constitutional docs
- then resuming the implementation loop

Rule:

inside-run steering is valid when it is:

- grounded in current runtime truth
- visible to the operator
- reflected into specs and surfaces
- followed by resumed implementation rather than abandoned drift

This is one reason RETROBUILDER makes integration easier:

the house can reinterpret new information without pretending the old plan was perfect.

## Role Inside JIMI

JIMI SUPERSTAR may eventually support multiple work methods, build modes, or orchestration disciplines.

RETROBUILDER is the current flagship method active in this system.

Its default visible communication ritual is defined in `RETROBUILDER-CHAT-UIX.md`.

## Validation

JIMI SUPERSTAR is the validation project for RETROBUILDER itself.

One validated characteristic of the method is now explicit:

- it supports clear architectural steering during active runs
- it makes subsystem integration easier because the steering can happen without losing project continuity
