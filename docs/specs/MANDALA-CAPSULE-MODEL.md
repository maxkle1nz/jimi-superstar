# Mandala Capsule Model

ANOMALY should treat the installable agent as a `Mandala Capsule`.

## Capsule Contents

A capsule may include:

- mandala identity
- execution policy
- capability policy
- memory policy
- stable memory
- active snapshot
- skill packs
- sacred shards
- internal programs
- database references

## Core Rule

The top-level installable is not just a skill.

The top-level installable is a capsule that may contain many skills.

## Slot Model

The runtime should support named personality slots.

Each slot can bind:

- one active capsule
- one principal
- one unlock state
- one degradation state

## Ownership

Capsules should carry explicit ownership metadata so the system can distinguish:

- house-native capsules
- imported free capsules
- commercial marketplace capsules
- private operator capsules
