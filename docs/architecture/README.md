# Architecture Overview

JIMI SUPERSTAR is organized around these major planes:

- Kernel Plane
- Truth Plane
- Capability Plane
- Compatibility Plane
- Execution Plane
- Memory Plane
- Surface Plane
- Closure Plane

Core runtime domains:

- Kernel Domain
- Truth Domain
- Capability Domain
- Execution Domain
- Memory Domain
- Interface Domain

The runtime is event-backed, policy-centered, and provider-neutral.
